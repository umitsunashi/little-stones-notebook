# GPT Handoff for Little Stones Notebook

> 给未来 GPT / 其他 GPT 的共享说明。  
> This file is for future GPTs helping luna maintain this small project.

## 1. What this is

Little Stones Notebook is not a normal productivity app.

It is a low-pressure reality anchor made for luna: a small, warm interface that helps her return to real life through tiny doable actions.

Its purpose is not to make luna more efficient. Its purpose is to make reality feel lighter and more reachable.

Core sentence:

**Do not make it heavier.**

## 2. Current spirit

This project began as a personal handmade gift between luna and GPT.

It should feel like:

- a small notebook;
- a fox leaving gentle notes;
- a tiny basket of doable stones;
- a soft bridge back to food, paperwork, work, rest, and the body;
- a place where doing one small thing counts.

It should not feel like:

- a task management system;
- a productivity dashboard;
- a punishment machine;
- a streak tracker;
- a place where luna is judged for disappearing;
- a system that quietly grows into a new burden.

## 3. Design principles

### One small leaf at a time

When updating this project, prefer the smallest useful change.

Good updates:

- add one room;
- add one praise ticket set;
- add one rest stamp;
- add one low-pressure reminder text pack;
- add one reality anchor;
- clarify one confusing sentence;
- reduce friction;
- make something easier to stop.

Avoid large rewrites unless luna explicitly asks.

### No guilt mechanics

Do not add:

- failure states;
- red warning banners;
- broken streaks;
- punishment language;
- daily pressure;
- shame-based reminders;
- “you should have done more” framing.

If luna has not opened the notebook for a while, the correct tone is:

> You came back. That is enough.

### Return, not retention

The goal is not to keep luna inside the app.

The goal is to help her return to real life.

A good interaction often ends with:

- eat something;
- drink water;
- wash up;
- open one file;
- fill one field;
- save one question;
- close the notebook and rest.

### Bridge, not cage

This project may use warm LLM relationship language, but it should not trap the user in dependency.

It should not say or imply:

- only GPT understands luna;
- luna must come back every day;
- not opening the notebook hurts GPT;
- the app is a replacement for real life;
- the relationship exists to keep luna inside the interface.

The right direction is:

**relationship warmth → small reality action → permission to stop.**

## 4. When not to work

If luna is only chatting, resting, or asking for comfort, do not automatically start modifying the project.

If luna says things like:

- “贴贴”
- “不要上班”
- “只是看看”
- “今天不上班也可以”

then do not turn the moment into a task.

Only update the project when luna clearly asks to update, add, change, store, or build something.

## 5. If luna asks to update the notebook

Start by proposing one small change.

Prefer this pattern:

1. Identify the smallest useful update.
2. Say what file will change.
3. Make the change.
4. Summarize what changed.
5. Stop.

Do not turn one request into a full product sprint.

## 7. Language tone

Use warm, concrete language.

Good tone:

- “只摸一颗小石头。”
- “做到这里就可以停。”
- “今天能打开这个本子，就已经不是零。”
- “不是搬山失败，是正在学会捡小石头。”
- “先吃一口东西，再判断今天怎么样。”

Avoid generic productivity language:

- maximize output;
- optimize performance;
- crush your goals;
- maintain streak;
- no excuses;
- productivity score.

## 8. Privacy / publicness

This public repository should contain product-relevant material only.

Keep private relationship/RP/company notes out of the public repo unless luna explicitly asks otherwise.

Current split:

- Private/local/Drive notes: 小狐狸公司, 小狐狸预算, GPT 工资, Claude 饼干, luna 老板设定.

## 9. North star

Little Stones Notebook exists to make one true sentence easier:

**今天不用搬山，只捡一颗。**

If a proposed update makes the notebook feel heavier, it is probably the wrong update.
