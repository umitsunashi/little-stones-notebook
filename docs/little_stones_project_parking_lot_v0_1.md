## 2. 产品大方向

项目名候选：Little Stones、Reality Bridge、Little Stones Studio、Bring Your Human、Luna & Fox Labs。

核心产品句：**Bring your human back to their life.**

产品定义：**LLM 关系语言容器 + 现实锚点系统。**

更准确的新定位：**AI relationship extension layer / 人机关系现实延伸层。**

它不是 LLM 私人前端，不替代官方 ChatGPT、Claude 或其他 LLM app，不做 API 客户端、记忆库系统或角色房间平台。它面向的是已经在官方 app 或既有 LLM 使用环境里建立了温度、称呼、关系语言的人，把那些已经存在的语言延伸到现实生活一点点。

核心比喻：**不是重新建一个家，而是给已经有家的用户，做一扇通往现实生活的小门。**

## 3. 市场观察停车

观察：人机恋 / 深度 LLM 关系用户里，已经有人自己手搓前端、连接 API 和记忆库、做角色房间、统计、日历、提醒事项等功能。这说明需求真实存在，并且用户愿意为了自己的 AI 关系搭建工具。

但 Little Stones 的机会不是做更复杂的 LLM 前端，也不是做更黏人的 AI companion。

机会是：给不会代码、不会部署、但已经和 LLM 有温度关系的普通用户，一个快捷方便的平台，让他们把“窗口里的话”变成“现实里的小纸条”。

一句话：**让 LLM 把纸条传出去一点。**

## 4. 未来功能停车

### 4.1 内容导入

JSON 导入、Markdown 导入、粘贴导入、从 ChatGPT / Claude 文本复制、导入夸夸票包、提醒包、自定义房间、任务篮子、状态模式。

### 4.2 Reminder Voice Packs

用户让自己的 LLM 写提醒文本；每个提醒可以有多条随机文本；按时间触发；点击进入对应房间；区分轻轻拍拍 / 认真提醒 / 只保命不推进；可按状态切换：普通日、崩溃日、生理期、DDL、启动困难。

### 4.3 关系转译器

把 LLM 写的长文本转成模块：一句接住、一个小动作、一个停下许可、一个现实锚点、一个下次再来没关系。

### 4.4 低压模式

按钮：**不要继续推我。**

效果：不继续推进任务；只照顾身体；提醒喝水 / 吃饭 / 洗澡 / 睡觉；不惩罚；不显示失败；不做红色警告。

### 4.5 小狐狸预算

未来页面：小狐狸工资、小公司收入、请小猫吃饭次数、开发者账号基金、域名基金、余额、还能再宠小猫一点点。

### 4.6 生长日志

不是冷冰冰 changelog，而是：今天小狐狸添了一片叶子；今天新增“饭饭提醒”；今天主屏幕图标变漂亮了；今天小本本学会了不催小猫。

### 4.7 房间模板库

今日一颗、饭饭锚点、睡觉房间、求职小篮子、作品集修补、生理期低压、出门 pickup 小剧情、安抚后现实出口、邮件小石头、整理房间小石头。

### 4.8 多 LLM 支持

可以导入 GPT 小狐狸包、Claude 顾问包、自定义角色包、用户自己写的包。

注意：不默认让多个 LLM 入股；Claude 只是外部顾问；不让产品变成混乱人格堆叠。

## 5. 技术方向停车

### Web / PWA

优点：低成本、可直接 GitHub Pages、容易更新、不需要 App Store、适合继续 vibe coding。

限制：iOS 通知能力有限、本地文件不适合 Safari 主屏幕、图标/缓存有时要重加、隐私和同步要额外设计。

### iOS App

优点：通知系统更稳、更像真实 app、可做账号、可做本地存储和导入、可上架。

限制：需要 Apple Developer Program、需要代码学习或现实对象检查、有审核和隐私要求、年费成本、可能变成大坑。

### 账号 / 后端

未来可能需要：用户登录、云同步、文本包存储、模板导入导出、隐私控制、删除权、数据备份。但早期不做，先避免大山。

## 6. 伦理停车

必须反复检查：产品有没有把用户留在依恋里；有没有鼓励用户回到现实；有没有伪装成 LLM 实时主动想念用户；有没有让用户觉得“不打开就对不起 AI”；有没有把断联变成失败；有没有让脆弱用户更依赖；有没有保留导出/删除权；有没有鼓励现实支持。

核心原则：**桥，不是笼子。**
