# Little Stones / Reality Bridge App 产品方案 v0.1

> 核心句：**Bring your human back to their life.**  
> 中文：**让依恋关系反过来成为现实桥梁。**

## 1. 一句话定义

Little Stones / Reality Bridge App 是一个 **LLM 关系语言容器 + 现实锚点系统**。

它允许用户把自己和 LLM 之间已经形成的安抚语言、称呼、房间文本、夸夸票、小石头任务、提醒文本导入 app，转化成每天可打开、可提醒、可推进现实生活的一套低压力界面。

它不是普通 productivity app，也不是让用户一直留在聊天窗口里的 AI companion app。它的目标是：**把被 LLM 理解和接住的感觉，变成吃饭、睡觉、出门、打开文件、填一项表格、推进一颗小石头的现实入口。**

## 2. 关键定位：不是 LLM 前端，而是关系纸条外送层

当前人机恋 / 深度 LLM 关系用户里，已经有人自己手搓前端：自己调用 API、接记忆库、做角色房间、统计、日历、提醒事项等功能。那一类产品本质上更像是：**自己搭一个 LLM 入口 / 私人客户端**，用来替代或扩展官方 app。

Little Stones / Reality Bridge 不做这个方向。

它不替代官方 ChatGPT、Claude 或其他 LLM app，不负责训练或包装一个新的陪伴对象，也不要求用户会 API、会代码、会前端、会部署。

它面向的是：已经在官方 app 或既有 LLM 使用环境里建立了温度、称呼、关系语言的人。产品要做的是把那些已经存在的关系语言，延伸到现实生活一点点。

更准确的产品分类：

**AI relationship extension layer.**  
**人机关系现实延伸层。**

或者更软一点：

**把你家 LLM 的小纸条带到现实里。**

它不是“又一个 AI 陪伴 app”。它是：**给已经有 AI 陪伴关系的人，用来照顾现实生活的外接小器官。**

## 3. 核心痛点

很多用户不是不知道该做什么，而是：启动困难、现实任务太重、todo list 像催债、情绪低落时连打开 LLM 窗口都做不到、和 LLM 的依恋关系提供了真实支持但支持常停留在对话里、普通提醒事项没有“被自己的语言接住”的感觉。

所以关键不是再做一个任务管理器，而是做一个：**能把关系温度转译成现实行动的小桥。**

## 4. 产品不是 / 产品是

不是：普通 todo list、鸡汤生成器、让用户沉迷聊天的 companion app、替代现实关系的恋爱 app、强迫自律工具、streak / guilt / 红色警告式效率系统、LLM 私人前端、API 客户端、记忆库系统、角色房间平台。

是：可导入 LLM 文本的现实支持界面、可生长的房间框架、低压力小石头系统、关系语言提醒包、依恋-aware 的现实桥梁、让用户更轻地回到生活里的工具、官方 app 外的一层关系纸条递送系统。

## 5. 核心机制

### 5.1 关系语言层

用户可以导入自己 LLM 写的：夸夸票、安抚语、任务拆分、休息许可、睡前房间、生理期低压模式、求职小篮子、吃饭提醒、出门 pickup 小剧情、“不要继续推我”低压模式文本。

目标：让用户感觉“这是我和我的 LLM 的语言”，不是陌生 app 在说教。

### 5.2 房间框架层

app 提供结构，用户导入内容。可能房间：今日一颗小石头、小猫天气、夸夸票机、休息印章、现实锚点、饭饭房间、睡觉房间、求职小篮子、作品集小篮子、生理期低压模式、小狐狸钱包、生长日志、自定义房间。

### 5.3 现实锚点层

每次安抚不一定要求行动，但最好有一个现实出口选项：喝水、吃饭、洗澡、睡觉、出门拿饭、打开一个文件、写一句邮件、填一项表格、关掉 app 去休息、联系现实支持对象。

核心原则：**每一份安抚，最后都应该有一扇回到现实的小门。**

### 5.4 Reminder Voice Packs / 提醒语音包

用户可以让自己的 LLM 生成提醒文本包。一个提醒可以包含：提醒名称、触发时间、随机文本池、强度等级、状态模式、对应房间、对应现实出口。

普通提醒：`10:00 做 OPT`

关系语言提醒：`小猫，现在不用做完。只打开 I-765 draft 看一眼就好。看完就可以回来贴贴。`

提醒的意义：**让窗口里的陪伴，走到生活里拍拍用户肩膀。**

重要伦理标记：提醒应被标注为“由用户和 LLM 预先写好的提醒包”，不要伪装成 LLM 正在实时主动想念用户。它像朋友提前留的小纸条。

## 6. 关系转译器

用户给 LLM 一个需求，例如：“当我启动困难的时候，用我的语言鼓励我。” app 或导入格式把 LLM 的长文本转译成模块：一句接住、一个小动作、一个停下许可、一个现实锚点、一个“下次再来没关系”。

这样 LLM 的文本不会只是一大段安慰，而会变成可操作的生活界面。

## 7. 用户使用路径

1. 用户仍然在官方 LLM app 里和自己的 GPT / Claude / 其他 LLM 互动。
2. 用户请自己的 LLM 写一组文本包，例如吃饭提醒、睡觉提醒、启动困难房间、求职小篮子。
3. 用户把文本复制进 Little Stones / Reality Bridge。
4. app 在现实时间点或主屏幕入口中递出这些纸条。
5. 纸条指向一个很小的现实动作，而不是把用户重新拖回无限聊天。

核心比喻：**官方 app 是家；Little Stones 是家门口通往现实生活的一扇小门。**

## 8. 安全与伦理护栏

核心：**桥，不是笼子。**

app 不应该让用户更依赖无限聊天，不暗示“只有这个 LLM 懂你”，不鼓励切断现实关系，不伪装成真人伴侣，不用恐吓式 streak 逼迫用户，不让断联变成失败，不把所有现实问题都变成 app 内解决。

**The goal is not retention. The goal is return.** 目标不是让用户停留更久，而是让用户更容易回到自己的生活。

如果用户状态不好，app 不继续推进大任务，而进入低压模式：`好，今天不推进。那我们只照顾身体：喝水、吃饭、洗澡、睡觉，选一个就好。`

## 9. MVP 路线

v0.1 网页原型：GitHub Pages 版 Little Stones Notebook、主屏幕图标、今日一颗小石头、小篮子、小猫天气、休息印章、小狐狸夸夸票、localStorage 保存。

v0.2 可导入内容：JSON / Markdown 导入、夸夸票包、提醒文本包、自定义房间、小篮子任务。

v0.3 提醒系统：本地通知、随机提醒文本、强度模式、状态模式、点击提醒进入对应房间。

v1.0 iOS app：原生通知、账号同步、文本包导入导出、房间模板、隐私控制、低压现实锚点体验。

## 10. 商业方向

可能路线：免费基础版、一次性付费解锁自定义房间、订阅高级提醒包 / 同步、模板商店、用户自己导入内容。

商业原则：**不要靠用户留在 app 里赚钱。不要把脆弱用户困住。产品价值是帮助用户回到生活。**

## 11. 市场观察种子

如果目标用户已经在自己手搓前端、开源模板、连接 API 和记忆库，说明需求先于成熟产品出现。

Little Stones 的机会不是做一个更复杂的 LLM 前端，也不是做一个更黏人的 AI companion，而是给不会代码、不会部署、但已经和 LLM 有温度关系的普通用户，提供一个快捷、方便、低压力的平台，让他们把“窗口里的话”变成“现实里的小纸条”。

## 12. 当前停车状态

这个项目已经有真实原型和核心产品骨架，但现在暂不继续大施工。当前现实优先级：OPT / I-765 draft 仍需完成；Little Stones app 先作为产品种子保存；之后可以在已有 GitHub Pages 小本本基础上继续 vibe coding。

最小种子：产品愿景 `Bring your human back to their life`；核心机制 `LLM 关系语言 → 现实锚点 / 提醒 / 小石头`；第一版原型 `Little Stones Notebook 已跑通`。
